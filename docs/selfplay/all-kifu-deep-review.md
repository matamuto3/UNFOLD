# 全棋譜 精密検討

作成: 2026-05-06

## 対象

`docs/selfplay` と `docs/experiments` 以下に残っている棋譜JSONを対象にした。

ただし、同じ対局を二重に数えないため、集計では以下を分けた。

| 種別 | 扱い |
| --- | --- |
| aggregate / 単体棋譜JSON | 一次棋譜として集計 |
| chunk JSON | aggregateに含まれるため重複扱い |
| kifu-light | aggregateから作った軽量派生のため重複扱い |
| snapshot | 進行中aggregateの複製なので重複扱い |
| summary / analysis / examples | 棋譜本体ではないため除外 |

一次棋譜として数えたものは229ファイル、23,373局。
重複・派生ファイルにも棋譜形式のものは1,199ファイル、36,978局ぶん存在したが、一次集計には足していない。

進行中の `original-depth3-max50-5000` は、レポート作成時点で130局完了していたため、その130局を一次棋譜として含めた。

## 全体結果

| 分類 | 局数 | 先手勝ち | 後手勝ち | 未決着 |
| --- | ---: | ---: | ---: | ---: |
| 全一次棋譜 | 23,373 | 12,897 | 8,887 | 1,589 |
| オリジナル駒 | 12,581 | 7,252 | 4,267 | 1,062 |
| 将棋駒 | 10,792 | 5,645 | 4,620 | 527 |

| 分類 | 先手勝率 | 後手勝率 | 未決着率 |
| --- | ---: | ---: | ---: |
| 全一次棋譜 | 55.2% | 38.0% | 6.8% |
| オリジナル駒 | 57.6% | 33.9% | 8.4% |
| 将棋駒 | 52.3% | 42.8% | 4.9% |

将棋駒モードの方が、履歴全体では先後差が小さい。オリジナル駒は先手が強く出ている。

ただし、ここには古いNPC・高速検証・煙テスト・ルール調整前後の棋譜が混ざっている。したがって、全体勝率は「歴史全体の傾向」であり、現在の厳密なバランス値ではない。

## 条件別の見方

| 条件 | 局数 | 先手勝率 | 後手勝率 | 未決着率 |
| --- | ---: | ---: | ---: | ---: |
| fast/bulk系 | 22,493 | 54.8% | 38.5% | 6.6% |
| legacy/条件不明 | 690 | 68.8% | 26.1% | 5.1% |
| strict no-fast/bulk | 190 | 45.3% | 22.1% | 32.6% |

現在の厳密条件に近い `fast=false / bulk=false` だけを見ると、先手勝率は45.3%まで下がるが、未決着が32.6%ある。これは「後手が勝っている」というより、「後手が序盤を止めているが、勝ち切れていない」と見るのが自然。

## 勝因

| 勝因 | 件数 | 率 |
| --- | ---: | ---: |
| 王の捕獲 | 18,546 | 79.3% |
| 本陣占領 | 3,238 | 13.9% |
| 未決着 | 1,589 | 6.8% |

このゲームの実戦的な勝ち筋は、現状では圧倒的に王の捕獲。相手本陣中心を上書きする勝ちより、王周辺の捕獲路を作る勝ちの方が多い。

したがってNPCの守備評価は、本陣上書きだけでなく「展界者の捕獲着地点」を第一優先で見る必要がある。

## 戦略別

| 戦略 | 局数 | 先手勝率 | 後手勝率 | 未決着率 |
| --- | ---: | ---: | ---: | ---: |
| attack / defense | 6,421 | 59.2% | 37.5% | 3.3% |
| attack / attack | 3,845 | 55.6% | 44.1% | 0.4% |
| balanced / balanced | 3,725 | 51.9% | 47.4% | 0.7% |
| defense / attack | 3,286 | 41.1% | 56.9% | 1.9% |
| defense / defense | 1,710 | 33.4% | 32.7% | 33.9% |

攻め対守りでも先手が勝ち越している。ただし、P1が防御、P2が攻撃の `defense / attack` では後手が勝ち越す。これは「後手だから弱い」というより、「能動的に勝ちに行く評価がある側が強い」ことを示している。

守り対守りは未決着が多すぎるので、定石学習には向きにくい。これは以前の直感どおり。

## 回収行動

全一次棋譜では、駒回収8,365回、展開図回収2,762回が出ている。

ただし、現在の厳密条件では駒回収2回、展開図回収4回しか出ていない。つまり、回収ルール自体が使えないのではなく、現在のNPC評価が回収をほぼ候補化できていない。

回収を定石化するなら、以下を評価に入れる。

| 行動 | 入れるべき評価 |
| --- | --- |
| 駒回収 | 自陣上の役割を終えた駒を持駒化し、王周辺の受け直しに使う |
| 駒回収 | 相手に取られるだけの駒を一度逃がす |
| 展開図回収 | 自分の展開図が相手の移動路になっている場合に消す |
| 展開図回収 | 空で上書きされていない展開図を持ち展開図化し、次の接続や受け直しに回す |

## 精密検討から見た定石仮説

1. 勝敗の中心は「王の捕獲」。
本陣上書きも重要だが、実戦では王捕獲が8割近い。

2. 後手の初期囲いは最左列10付近に受けの帯を作る。
中央へ伸ばしすぎると先手の通路になる。引きすぎると王周辺だけが孤立する。

3. 後手は駒単体より配置形が重要。
強い駒を置くより、王周辺に相手の最終着地点を作らないことが大事。

4. 先手のジェイ系、ティー、クロス、ソーは警戒対象。
これらは早期の王捕獲や本陣占領につながりやすい。

5. 守りNPCは勝ち切りが弱い。
厳密条件では未決着が多く、後手が受け切っても勝ちへ転換できていない。

6. 回収はまだAIが使えていない。
履歴全体では回収が出ているが、現在条件ではほぼ出ない。守備定石として育てる余地が大きい。

## NPCへ反映する評価項目

| 優先 | 評価項目 | 内容 |
| --- | --- | --- |
| 高 | 王周辺の捕獲着地点 | 自分の展界者が次手で取られるマスを大減点 |
| 高 | 受けの帯 | 後手初期展開は最左列10付近を加点 |
| 高 | 自展開図の敵通路化 | 自分の展開図が敵の移動路になれば減点 |
| 中 | 取り返し | 敵が王周辺に来ても次に取れるなら加点 |
| 中 | 危険初手対応 | ジェイ系、ティー、クロス、ソーへの専用受け |
| 中 | 回収候補 | 通路化した展開図や役割終了駒の回収を候補化 |
| 低 | 駒単体補正 | 駒の強弱より配置を優先し、補正は控えめにする |

## 次の進め方

まずは現在進行中の `original-depth3-max50-5000` を100戦単位で更新し、この全棋譜レポートの「厳密条件」部分を育てる。
全履歴は傾向を見るために使い、現在のバランス判断は `fast=false / bulk=false` の棋譜を主軸にする。

次に実装すべきは、後手NPCの「王周辺の捕獲着地点評価」と「受けの帯評価」。この2つが入ると、序盤即死のかなりの部分を減らせる可能性が高い。
