# UNFOLD 20戦区切り自己対局メモ

検証条件:

- 先手: 攻めNPC
- 後手: 守りNPC
- 1ブロック: 20戦
- 最大手数: 240 ply
- 目的: 山札消費そのものではなく、先手の攻め成功率と後手の守備成功率を近づける定石を探す
- 集計方針: 未決着は、後手が手数上限まで守り切った「後手守備成功」として扱う

## 攻めの定石候補

- 接続点は1本ではなく複数作る。1本だけだと妨害や上書きで止まりやすい。
- 王捕獲だけでなく、相手本陣中心を展開図で上書きできる形を常に見る。
- 攻撃駒を単独で突っ込ませるより、界騎士・結界士・護衛士系で足場を作ってから圧をかける。
- 攻め側の回収は、相手本陣近くの主力駒が取られそうなときに限定する。攻めの手番を失うため乱用しない。

## 守りの定石候補

- 初期スタンバイは、各プレイヤーが最初の手札3枚から対応駒を本陣に置き、選んだ展開図を持ち展開図にする現行ルールを前提にする。
- オリジナル版では「堅守2 / 妨害1 / 反撃0」が平均78.8手まで伸びた。長期戦化の中心候補。
- オリジナル版では「堅守2 / 妨害0 / 反撃1」が後手7勝2敗で勝率が高い。ただし平均13.4手で、短期反撃寄り。
- 将棋駒版では「堅守1 / 妨害0 / 反撃2」が後手7勝6敗で有望。受け一辺倒より反撃を混ぜるほうが押し返せる可能性がある。
- 将棋駒版の「堅守0 / 妨害2 / 反撃1」は後手3勝28敗でかなり悪い。妨害だけでは本陣中心を守れない。

## 回収の使いどころ

- 駒回収は「取られそうな駒を逃がす」用途が最も実戦的。オリジナル100戦では駒回収319回中192回が被攻撃状態、将棋合計200戦では192回中119回が被攻撃状態だった。
- 守り側の駒回収は、遠くで孤立した駒を持駒に戻し、自陣に打ち直す準備として強い。
- 展開図回収は、空で上書きされていない展開図を持ち展開図化し、接続や受け直しに変換できる局面で価値が高い。
- ただし、回収は1手を使う。即負けの本陣上書きや王捕獲が迫っているときは、回収より受けを優先する。

## 次の調整仮説

- 将棋駒版は先手有利が大きいので、後手NPCの初期スタンバイで「本陣中心の防御点」と「相手接続点の遮断」をさらに重くする。
- 回収が発生したブロックは長期戦化しやすいため、後手NPCは被攻撃駒の回収と、安全な展開図の持ち展開図化を早めに判断する。
- オリジナル版は攻守差4ptまで寄っているので、ここからは過度に後手を強化しすぎず、短期決着パターンだけを狙って潰す。
